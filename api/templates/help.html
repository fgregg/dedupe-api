Field types

Dedupe can compare your fields in different ways depending on the makeup of the data. For more details, see the dedupe library documentation on field definitions.

Field type Description Good for
Text Default field type. Fields will be compared based on how similar they are to each other, character by character. Person or company names
Short text If the data in this field is shorter than 10 characters, using short text can be used to speed up the matching performance. Zip codes and city names
Long text Compares entire words. Useful for longer text fields. Product descriptions or article abstracts
Exact match? Checks to see if the fields exaclty match or not. Cleaned and consistent data
Exists? Measures whether both, one, or neither of the fields are defined. Sparsely populated data (when presence is significant)
Address Splits addresses in to separate components using usaddress. Addresses in the United States
Price Measures the difference between two prices in the same currency. Price amounts in the same currency
Latitude/Longitude Compares fields based on geographic proximity via the Haversine Formula. Latitude and Longitude pairs
Set (advanced) Used for comparing short lists of pre-defined elements. Keywords or categories
Categories (advanced) Used for comparing long lists of pre-defined elements. Keywords or categories
